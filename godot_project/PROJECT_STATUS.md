## 📐 架构评估

### 优点 ✅
1. **数据与视觉完全解耦** — 导航网络是纯数据对象，不挂载 GameObject/Node，序列化和测试都很友好
2. **清晰的分层** — Core (数据) → Systems (逻辑) → UI (表现) 层次分明
3. **完整的建造交互** — 预览 ghost、吸附、取消、多楼层，交互手感已经比较完善
4. **网络可视化** — 按 N 切换可视化，按 M 输出拓扑图，调试体验很好
5. **自动跨结构连接** — 新建的滑行道/停机位 30m 内自动连接到已有网络
6. **BFS 连通性检测** — 能实时检测网络是否有孤立分区

### 关注点 ⚠️
1. **`_is_just_clicked()` 重复实现** — 每个 Builder 都手动实现了帧去重，应该提取到 `BaseBuilder`
2. **Autoload 之间耦合** — `ATCManager` 直接访问 `AirportManager.active_runways`，`RunwayNetworkManager` 和 `AirportManager` 功能有重叠
3. **节点搜索性能** — gate 吸附和自动连接都是 O(N) 遍历，节点 >1000 会有问题（代码中已有注释标注）
4. **资产目录为空** — `assets/` 目录没有任何文件

4. **存档系统** — 将 `all_runways` + `all_gates` JSON 序列化到文件
5. **经济系统** — 基础资金 + 建造花费 + 航班收入
